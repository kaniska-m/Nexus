Multi-Agent Pipeline

6 specialized agents in a conditional LangGraph DAG — not a chain

1
Orchestrator

Reads industry, generates dynamic compliance checklist, assigns tasks, detects stalls, escalates failures — no hardcoding required.

2
Collector

Smart form generation, tracks submitted vs pending, auto-sends contextual reminders tailored to exactly what's missing per supplier.

3
Verifier

OCR reads PDFs, checks expiry dates and cert numbers, cross-references MCA21, GSTN, CDSCO APIs, detects fraud signals.

4
Risk Scorer

Sanction screening via OpenSanctions, risk scoring Low/Med/High, LLM writes human-readable risk rationale narrative.

5
Audit Agent

Compiles all decisions with timestamps, generates regulator-ready PDF audit pack in compliant legal language.

6
Monitor

Post-approval health surveillance: cert expiry, risk drift, SLA performance, continuous sanction screening. Permanent compliance.

Conditional DAG edges: Each agent communicates through shared state — Fraud → HALT · Stall → Escalate · Missing docs → Retry

Stall Detection

Orchestrator detects when Collector hasn't progressed. Retries with escalated reminders before flagging to human.

Fraud HALT

Verifier flags suspicious docs → pipeline halts immediately. Human receives halt report. No silent failures.

API Failures

Gov API errors → agents retry → log failure → pipeline continues with available data, gap flagged.

Ambiguous Input

Risk Scorer can't determine High/Low → generates uncertainty flag with partial reasoning rather than guessing.

NexusStateManager — Shared State

Thread-safe single source of truth for all 6 agents. Agents read/write through manager — no direct coupling. State includes: checklist, doc status, risk score, audit log.

Responsibilities Split

Each agent has one clear job. Orchestrator never does document reading. Verifier never scores risk. Zero bleed.

Structured Handoffs

Agents communicate via shared state — not direct calls. Output of one becomes input of next automatically.

Orchestration Pattern

Orchestrator = conductor. Conditional edges determine routing. No agent decides its own successor.

Isolation = Testability

172 tests: each agent unit-tested independently. Integration tests cover the full DAG end-to-end.

LLM Routing for Cost Efficiency
